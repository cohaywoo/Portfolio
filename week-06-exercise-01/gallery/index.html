This Gallery page is located inside the gallery/ folder, similar to how the Projects page is structured. It demonstrates the same principles: nested folders, relative paths, and shared CSS styling.

The Three Navigation Patterns

Throughout this exercise, you've encountered three different navigation styles:

1. Blocky Navigation (Home Page)

The first pattern uses a wood-themed design with a grass background. Links are styled as wooden blocks with gold text. Hover effects lift the blocks upward with a smooth animation.

2. Stone Steps Navigation (About Page)

The second pattern creates a 3D stepped effect using inset shadows and border styling. It has an earthy dirt/stone color scheme. Hovering reveals diamond-colored highlights and a lifted effect.

3. Diamond Luxury Navigation (Projects & Gallery)

The third pattern centers the navigation items with a diamond blue color scheme. The dark background creates high contrast, and the centered layout makes it feel more luxurious and polished. Hover effects scale upward with a shadow.

Consistent CSS Linking

Just like the Projects pages, the Gallery uses ../css/style.css to link to the shared stylesheet. This is the key to maintaining a consistent design across all pages.

Complete Your Learning

You've now explored:

  • Root-level pages (Home, About) using direct file links
  • Nested pages (Projects, Gallery) using relative ../ paths
  • Multiple pages within nested folders (project-detail.html)
  • Three distinct navigation styles and hover effects
  • Minecraft-themed design and color schemes

Summary of Key Concepts

Relative Paths: Always remember ../ goes up one folder level. Use it when linking from nested pages back to the root.

Shared Resources: CSS files in a central location can be accessed from all pages using the correct relative path.

Navigation Flexibility: You can style navigation differently on each page while maintaining functional links.

Testing: Always click through your entire site to ensure all links work and CSS loads correctly.
